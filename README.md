# SaaS Support & Troubleshooting Lab

## Overview
This project demonstrates hands-on troubleshooting of real-world SaaS-style support incidents in a controlled lab environment.

Each scenario is structured as a support case, focusing on:
- Identifying user-reported issues
- Performing structured technical investigation
- Determining root cause
- Implementing and validating resolutions

The goal is to simulate the day-to-day work of a Technical Support Engineer or Service Desk Analyst.

---

## Scenarios


1. [Application Access Failure (DNS Misconfiguration)](scenarios/01-application-access-dns.md)  
   Diagnosing application access issues caused by DNS resolution failures.

2. [Shared Resource Access Issue (Active Directory Permissions)](scenarios/02-permissions-group-access.md)  
   Troubleshooting access denial due to group membership and permission configuration.

3. [Workstation Performance Degradation (Resource Utilization)](scenarios/03-performance-degradation.md)  
   Identifying and resolving system performance issues caused by resource-heavy processes.

## Skills Demonstrated

## Why This Project

This lab focuses on practical, real-world support scenarios rather than theoretical configuration.

Each case reflects common issues handled in:
- SaaS support environments
- IT service desks
- Technical support engineering roles

The emphasis is on **how problems are investigated and resolved**, not just the final fix.
